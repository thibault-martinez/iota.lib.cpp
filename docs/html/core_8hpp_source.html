91  * Find the transactions which match the specified input and return. All input values are lists,
92  * for which a list of return values (transaction hashes), in the same order, is returned for

107  /*
108  * Get the inclusion states of a set of transactions. This is for determining if a transaction
109  * was accepted and confirmed by the network or not. You can search for multiple tips (and thus,
110  * milestones) to get past inclusion states of transactions.
111  * This API call simply returns a list of boolean values in the same order as the transaction

113  * https://iota.readme.io/docs/getinclusionstates

117  /*
118  * Similar to getInclusionStates. It returns the confirmed balance which a list of addresses
119  * have at the latest confirmed milestone. In addition to the balances, it also returns the
120  * milestone as well as the index with which the confirmed balance was determined. The balances
121  * is returned as a list in the same order as the addresses were provided as input.

124  Responses::GetBalances getBalances(const std::vector<std::string>& addresses,

126  /*
127  * Tip selection which returns trunkTransaction and branchTransaction. The input value is depth,
128  * which basically determines how many bundles to go back to for finding the transactions to
129  * approve. The higher your depth value, the more "babysitting" you do for the network (as you
130  * have to confirm more transactions).
131  * https://iota.readme.io/docs/gettransactionstoapprove
132  */
133  Responses::GetTransactionsToApprove getTransactionsToApprove(const int& depth) const;
134  /*
135  * Attaches the specified transactions (trytes) to the Tangle by doing Proof of Work. You need
136  * to supply branchTransaction as well as trunkTransaction (basically the tips which you're
137  * going to validate and reference with this transaction) - both of which you'll get through the
138  * getTransactionsToApprove API call.
139  * The returned value is a different set of tryte values which you can input into
140  * broadcastTransactions and storeTransactions. The returned tryte value, the last 243 trytes
141  * basically consist of the: trunkTransaction + branchTransaction + nonce. These are valid
142  * trytes which are then accepted by the network. https://iota.readme.io/docs/attachtotangle
143  */
144  Responses::AttachToTangle attachToTangle(const std::string& trunkTransaction,

146  const int& minWeightMagnitude,
147  const std::vector<std::string>& trytes) const;

151  */
152  Responses::InterruptAttachingToTangle interruptAttachingToTangle() const;
