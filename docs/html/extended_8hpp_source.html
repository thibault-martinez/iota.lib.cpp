182  /*
183  * Main purpose of this function is to get an array of transfer objects as input, and then prepare
184  * the transfer by generating the correct bundle, as well as choosing and signing the inputs if
185  * necessary (if it's a value transfer). The output of this function is an array of the raw
186  * transaction data (trytes).
187  *
188  * @param seed 81-tryte encoded address of recipient.
189  * @param security The security level of private key / seed.
190  * @param transfers Array of transfer objects.
191  * @param remainder If defined, this address will be used for sending the remainder value (of the
192  * inputs) to.
193  * @param inputs The inputs.
194  * @return Returns bundle trytes.
195  */

202  /*
203  * This function returns the bundle which is associated with a transaction. Input can by any type
204  * of transaction (tail and non-tail). If there are multiple bundles (because of a replay for
205  * example), it will return multiple bundles. It also does important validation checking
206  * (signatures, sum, order) to ensure that the correct bundle is returned.
